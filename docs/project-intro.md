# GenUI Architect：说句话就能生成 UI 的工具

你在输入框打一句"做个赛博朋克仪表盘"，几秒后屏幕上就出现了一个完整的仪表盘界面——有数据卡片、有图表、有表格，还能点击交互。这就是 GenUI Architect 干的事。

下面讲讲它是怎么实现的。不需要你懂 React 或 AI，我尽量用大白话说。

---

## 它到底在干嘛？

简单说就三步：

1. **你说话** → "做个登录页面"
2. **AI 想** → 把你的话翻译成一棵 JSON 结构（可以理解为 UI 的"蓝图"）
3. **浏览器画** → 把这份蓝图渲染成真实的界面

而且不是等 AI 想完才画——它一边吐 JSON，浏览器一边画，你能看到界面像拼图一样一块块"长"出来。

---

## 为什么用 JSON 当蓝图？

你可能会问：为什么不直接让 AI 写代码？

因为代码太容易出错。少一个括号整个页面就白屏了。但 JSON 不一样——结构简单、可预测、出了问题还能修。

每个 UI 组件就是一个 JSON 节点：

```json
{
  "card": {
    "title": "今日收入",
    "children": [
      { "stat": { "value": "$12,400", "trend": "+8%" } }
    ]
  }
}
```

一个 key（比如 `card`）代表组件类型，value 就是这个组件的配置。嵌套 `children` 就是子组件。整个 UI 就是这样一棵树。

---

## 五个核心能力

### 1. 流式渲染——一边生成一边看

AI 每吐出一小段 JSON，系统就尝试修复它（比如 `{"card":{"title":"He` 还没写完，系统会自动补上 `"}}` 让它变成合法 JSON），然后立刻渲染出来。

你看到的效果是：界面从上到下逐渐成形，而不是白屏干等十几秒。

### 2. 自愈——画崩了自己修

AI 生成的东西不一定完美，有时候 JSON 结构有问题，组件渲染会崩。

这时候系统会自动把崩掉的那块 JSON + 错误信息发回给 AI："这段让我崩了，帮我修一下。"AI 修好后替换回去，用户可能都没察觉出过问题。

### 3. Tool Call——能查真实数据

AI 不只是凭空画图，还能调用工具获取真实数据。

比如你说"显示比特币价格"，AI 会先调 CoinGecko API 拿到实时价格，再用这个数据生成 UI。目前有这些工具：

- **天气查询**（Open-Meteo 真实 API）
- **加密货币价格**（CoinGecko 真实 API）
- **业务操作**（发邮件、排会议、加购物车等，目前是模拟的）
- **实用工具**（贷款计算、货币转换、翻译等）

想加新工具也很简单：写个处理函数，注册一下，告诉 AI 有这个工具就行。

### 4. 编辑模式——生成完还能改

UI 生成完不是终点。切到 Design 模式后：

- **点哪改哪**：点击界面上的任何组件，它会被选中高亮
- **自然语言精化**：选中一个卡片后输入"改成深色背景加渐变"，AI 只改这一个组件，其他不动
- **属性面板**：右侧弹出 Inspector，可以直接改组件的属性值（标题、颜色、布局等）
- **一键变体**：选中组件后点 Make Variation，AI 基于当前组件生成一个风格变体

底层原理是路径操作——系统知道你点的组件在 JSON 树里的哪个位置（比如 `root.container.children.2.card`），把那个子树取出来发给 AI 改，改完放回去。

而且支持 **Ctrl+Z 撤销**，最多回退 50 步，改坏了随时回来。

### 5. 代码导出——拿走就能用

生成的 UI 满意了，点 Export Code，系统把整棵 JSON 树转成可用的 React JSX 代码，弹窗展示出来，一键复制到剪贴板，粘贴到你自己的项目里就能跑。

## 三家 AI 随便切

不绑定任何一家 AI 服务商。打开设置面板，三个选项卡：

- **Gemini**：Google 的，生成速度快，而且是唯一能生成图片的（VN 模式要用）
- **OpenAI**：GPT 系列，通过 JSON Schema 约束输出结构最严格
- **Anthropic**：Claude 系列，推理能力强，生成的 UI 结构通常更合理

每家都可以填自己的 API Key 和自定义地址（比如公司内网的模型服务、或者局域网里自己跑的模型）。切换后立刻生效，不用重启。

自定义地址不用担心跨域问题——系统内置了一个代理，浏览器的请求先发到自己的服务器，服务器再转发到你的模型地址。

## 29 个 UI 积木

AI 能生成的组件取决于系统里有多少种"积木块"。目前有 29 种，覆盖了最常见的 UI 场景：

| 类别 | 组件 |
|------|------|
| 布局 | 容器、英雄区、分栏、Bento 网格 |
| 表单 | 按钮、输入框、文本域、开关、滑块、日历 |
| 展示 | 文字、数据卡片、进度条、提醒、头像、标签、分割线、图片 |
| 复合 | 卡片、表格、图表、手风琴、地图、看板、标签页、步骤条、时间线、代码块 |
| 特殊 | 视觉小说舞台（后面细说） |

重的组件（图表、地图、表格）用了懒加载——你不用就不下载，页面不会白白变大。

想加新积木也有套路，5 步走：创建组件 → 写验证规则 → 注册到系统 → 告诉 AI 有这个组件 → 写测试。这 5 步漏了任何一步都会出问题（验证不认识 → 渲染空白 → AI 不知道有它）。

## Galgame 模式——是的，你没看错

除了正经的 UI 生成，这个项目还内置了一个**视觉小说引擎**。

切到 VN Mode 后，AI 不再生成按钮和表格，而是生成一个三层的舞台：

- **背景层**：电影感的动画背景，带暗角特效
- **角色层**：角色立绘站在舞台上，按位置（左/右/中/特写）自动排列，支持弹跳、摇晃、滑入等动画
- **对话层**：打字机效果逐字显示台词，带说话者名牌

出现选项时，背景模糊，选项浮现，每种选项有不同颜色风格（攻击性的是红色、浪漫的是粉色、普通的是青色）。

没选项时点击画面就是"继续"，AI 会接着生成下一段剧情。

角色的头像和背景图可以 AI 实时生成（Gemini provider 支持），也可以用外部图片链接。

## 工作区长什么样

打开页面你会看到三栏布局：

**左边：聊天栏**
- 上面是对话历史（你说的话 + AI 生成的结果）

- 一个设备框架（可以切桌面/手机视图）

- 编辑模式下才出现
- 显示选中组件的所有属性
- 可以直接修改

顶部工具栏有：设备切换、撤销/重做、角色切换（admin/user 看到不同 UI）、代码导出。

## 一些你可能好奇的问题

**AI 生成的 JSON 格式不对怎么办？**
系统有两道保险：第一道是 Zod 验证（生成完检查结构），第二道是自愈（崩了让 AI 修）。流式传输期间的不完整 JSON 不算错，系统会自动跳过。

**支持 undo 吗？**
支持，最多 50 步，Ctrl+Z / Ctrl+Y。每次生成、编辑都会记录历史。

**API Key 安全吗？**
这是纯前端项目，key 存在浏览器的 localStorage 里，请求直接从浏览器发出。适合个人和内网使用，不适合公开部署。

**怎么加新的 AI 工具（tool）？**
四步：写处理函数并注册 → 在 index 里导入 → 在系统指令里告诉 AI 这个工具的用法 → 写测试。

**怎么加新的 UI 组件？**
五步：写组件 → 写 Zod 验证规则并加入总 schema → 注册到 Registry → 在 COMPONENT_SPECS 里描述它 → 写测试。

## 技术栈一览

- **前端**：React 19 + TypeScript + Tailwind CSS + Framer Motion
- **AI SDK**：@google/genai + openai + @anthropic-ai/sdk
- **验证**：Zod（运行时）+ JSON Schema（约束 AI 输出）
- **测试**：Vitest（215 个单元测试）+ Playwright（E2E）
- **构建**：Vite + pnpm
- **CI**：GitHub Actions（类型检查 → 单元测试 → 构建 → E2E）
